WHAT MAKES AI RUN ITSELF

An AI model on its own is just very good autocomplete — it guesses the next word. No memory, no actions, no decisions. An "entity" is everything you wire around it so it can remember, act, decide, and improve. This is the map of that wiring.

THE ONE IDEA   watch what it does → freeze the parts that repeat into code → shrink the cost to almost nothing. Make the hidden parts visible, then drive the waste toward zero.

five questions that place any AI

PATH — who decides the next step — you, or it?
MULTIPLICITY — one AI, or several working together?
ABSTRACTION — what can it reach beyond its own words — memory? tools? writing its own tools?
pROMPTING — how is it told what to do?
aSYNC — does it wait in lockstep, or run on its own clock?
each question has levels, from "barely" to "fully on its own." The gold dot is the highest level actually proven — not just hoped for.

ten parts it can't live without

take any one away and the whole thing falls apart:
somewhere to run · a clear goal · turning repeated work into code · falling back to the model when something's new · building its own next version · knowing what it's made of · sensing when it's stuck · a way past the wall · lines it must never cross · a path to roll back

what it's always doing

build the setup · pass results along · watch itself the whole time

the loop you run around it

design it · run it on a clock you can watch · ship it · learn from it

the three rules it lives by

let behaviour emerge, don't force it · always be able to roll back · always be able to see what it did
